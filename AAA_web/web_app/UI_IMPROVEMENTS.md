### 구현 방법

모든 뒤로가기 버튼은 **동일한 방식**으로 구현:

```typescript
import { ArrowBack as ArrowBackIcon } from '@mui/icons-material';
import { useNavigate } from 'react-router-dom';

const navigate = useNavigate();

<IconButton
  onClick={() => navigate('/chat')}
  sx={{
    color: 'primary.main',
    '&:hover': {
      bgcolor: 'rgba(0, 0, 0, 0.04)',
    },
  }}
>
  <ArrowBackIcon />
</IconButton>
```

**동작**: 클릭 시 `/chat` (채팅 홈 페이지)으로 이동

## 2. 휴가 관리 데스크톱 UI 레이아웃 개선

### 문제점
- **내 휴가 현황** 카드: 내부 요소들이 잘림
- **결재진행 현황** 카드: 내부 요소들이 잘림
- 카드 높이가 `102px`로 고정되어 있어 내용이 넘침

### ✅ 해결 방법

**파일**: `web_app/src/components/leave/DesktopLeaveManagement.tsx` (라인 305-460)

#### 변경 사항

| 항목 | Before | After |
|------|--------|-------|
| **카드 높이** | `height: 102` | `minHeight: 140` |
| **CardContent 패딩** | `p: 1` | `p: 2` |
| **아이콘 크기** | `fontSize: 16` | `fontSize: 18` |
| **타이틀 폰트 크기** | `fontSize: '14px'` | `fontSize: '15px'` |
| **내부 요소 간격** | `gap: 1, mt: 1` | `gap: 1.5, mt: 2` |
| **휴가 종류 박스 패딩** | `p: 0.5` | `p: 1.5` |
| **휴가 종류 폰트** | `fontSize: '10px'` | `fontSize: '11px'` |
| **잔여일 폰트** | `fontSize: '16px'` | `fontSize: '18px'` |
| **결재 현황 박스 패딩** | `p: 0.75` | `p: 1.5` |
| **결재 현황 숫자 폰트** | `fontSize: '20px'` | `fontSize: '22px'` |

## 3. 적용된 디자인 개선

### 3.1 간격 및 여백 최적화
- **카드 간격**: `gap: 2` (16px)
- **내부 패딩**: `p: 2` (16px)
- **요소 간 간격**: `gap: 1.5` (12px)
- **상하 여백**: `mt: 2, mb: 2` (16px)

### 3.2 폰트 크기 개선
- **타이틀**: 14px → 15px
- **잔여일 숫자**: 16px → 18px (강조)
- **결재 현황 숫자**: 20px → 22px (강조)
- **휴가 종류/라벨**: 10px → 11px (가독성 향상)

### 3.3 박스 패딩 증가
- **휴가 종류 박스**: 0.5 → 1.5 (12px)
- **결재 현황 박스**: 0.75 → 1.5 (12px)
- **Chip**: height 24 → 26, px 추가

## 4. 반응형 동작

### 데스크톱 (≥900px)
- 뒤로가기 버튼 표시
- 개선된 레이아웃으로 모든 요소 표시
- 카드가 자동으로 높이 조정 (`minHeight: 140`)

### 모바일 (<900px)
- MobileMainLayout 사용 (햄버거 메뉴)
- 뒤로가기 버튼 헤더에 표시
- 기존 모바일 UI 유지

## 5. 테스트 방법

### 5.1 뒤로가기 버튼 테스트
```bash
cd web_app
npm run dev

# 브라우저에서 테스트:
1. /leave (휴가 관리) → 뒤로가기 버튼 클릭 → /chat 이동 확인
2. /gift (받은선물함) → 뒤로가기 버튼 클릭 → /chat 이동 확인
3. /settings (환경 설정) → 뒤로가기 버튼 클릭 → /chat 이동 확인
4. /approval (전자결재) → 뒤로가기 버튼 클릭 → /chat 이동 확인
```

### 5.2 휴가 관리 UI 개선 테스트
```bash
# 브라우저를 1280px 이상으로 설정
1. /leave 페이지 접속
2. 내 휴가 현황 카드 확인:
   - 휴가 종류 4개 모두 표시되는지
   - 잔여일/총일수가 잘리지 않는지
   - 여백이 충분한지
3. 결재진행 현황 카드 확인:
   - 대기중/승인됨/반려됨 모두 표시되는지
   - 숫자가 잘리지 않는지
   - 아이콘과 텍스트가 정렬되는지
```

## 7. 주요 개선 포인트

### ✅ 사용자 경험 개선
1. **네비게이션 개선**: 모든 페이지에서 홈으로 쉽게 돌아갈 수 있음
2. **UI 가독성 향상**: 휴가 관리 화면의 정보가 명확하게 표시됨
3. **일관성 향상**: 모든 페이지에 동일한 스타일의 뒤로가기 버튼

### ✅ 디자인 개선
1. **간격 최적화**: 요소 간 충분한 여백 확보
2. **폰트 크기 조정**: 중요 정보 강조
3. **레이아웃 유연성**: `minHeight` 사용으로 내용에 따라 자동 조정

### ✅ 코드 품질
1. **재사용 가능한 패턴**: 모든 뒤로가기 버튼이 동일한 방식으로 구현
2. **일관된 스타일링**: MUI sx props 사용
3. **접근성 고려**: IconButton with ARIA labels
